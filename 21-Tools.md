# Chapter 21: Tools

This chapter provides a comprehensive guide to the essential tools used in Active Directory security assessments. For each tool, we cover its purpose, installation, key features, usage examples, advantages, limitations, and detection considerations.

## BloodHound

### Purpose
BloodHound is a tool that uses graph theory to reveal the hidden and often unintended relationships within an Active Directory environment. It is used to visualize attack paths and identify privilege escalation opportunities.

### Installation
- **Data Collector (SharpHound)**: Download the latest release from GitHub or use the PowerShell script.
- **Analyzer (BloodHound GUI)**: Download the GUI from GitHub and install the required dependencies (Neo4j, .NET).

### Features
- Visualizes complex AD relationships.
- Identifies shortest paths to high-privileged targets.
- Provides pre-built and custom Cypher queries.
- Supports marking nodes as owned to trace attack paths.

### Usage
```powershell
# PowerShell (SharpHound.ps1)
Invoke-BloodHound -CollectionMethod All -Domain contoso.com -OutputDirectory C:\Temp\BloodHound
```

### Advantages
- Powerful visualization of attack paths.
- Highly customizable with Cypher queries.
- Essential for understanding complex AD environments.

### Limitations
- Requires initial access to the domain to collect data.
- Data collection can be noisy and easily detected.
- Steep learning curve for advanced queries.

### Detection Considerations
- Monitor for LDAP queries targeting sensitive attributes (e.g., `msDS-AllowedToDelegateTo`, `servicePrincipalName`).
- Look for the execution of SharpHound binaries or scripts.

## SharpHound

### Purpose
SharpHound is the official data collector for BloodHound. It gathers data about domain trusts, group memberships, ACLs, and sessions.

### Installation
- Download the latest release from GitHub.

### Features
- Comprehensive data collection (users, groups, computers, ACLs, sessions).
- Multiple collection methods (e.g., All, Default, Group, Session).
- Outputs data in JSON format for BloodHound analysis.

### Usage
```bash
# Using SharpHound.exe
SharpHound.exe --collectionmethods All --domain contoso.com --outputdirectory C:\Temp\BloodHound
```

### Advantages
- Official and well-maintained data collector.
- Fast and efficient data collection.
- Supports various collection methods for targeted analysis.

### Limitations
- Generates significant network traffic and LDAP queries.
- Easily detected by EDR and SIEM solutions.
- Requires execution on a domain-joined machine.

### Detection Considerations
- Monitor for high volumes of LDAP queries from a single source.
- Look for the creation of specific files (e.g., `BloodHound.bin`) in temp directories.

## PowerView

### Purpose
PowerView is a PowerShell tool to gain network situational awareness on Windows domains. It provides functions for enumerating domains, trusts, users, groups, and more.

### Installation
- Download the `PowerView.ps1` script from GitHub.

### Features
- Comprehensive domain enumeration capabilities.
- ACL and permission analysis.
- Delegation and trust enumeration.
- GPO analysis.

### Usage
```powershell
# PowerShell
. .\PowerView.ps1
Get-DomainController
Get-DomainUser -Identity Administrator
Get-DomainGroup -Identity "Domain Admins"
Get-DomainComputer -Properties dnshostname, operatingsystem
```

### Advantages
- Powerful and flexible PowerShell-based enumeration.
- No external dependencies (other than PowerShell).
- Can be loaded directly into memory to avoid writing to disk.

### Limitations
- PowerShell-based, making it susceptible to AMSI and logging.
- Can be noisy if not used carefully.
- Requires knowledge of PowerShell and AD concepts.

### Detection Considerations
- Monitor for PowerShell script block logging (Event ID 4104).
- Look for the loading of PowerView functions or modules.
- Monitor for LDAP queries generated by PowerView commands.

## Rubeus

### Purpose
Rubeus is a C# framework for raw Kerberos interaction and abuse. It is used for Kerberoasting, AS-REP Roasting, ticket manipulation, and delegation attacks.

### Installation
- Download the latest release from GitHub or compile the source code using Visual Studio.

### Features
- Kerberoasting and AS-REP Roasting.
- Ticket manipulation (Golden Ticket, Silver Ticket, Diamond Ticket).
- Delegation abuse (S4U2Self, S4U2Proxy).
- Pass-the-Ticket and Overpass-the-Hash.

### Usage
```powershell
# PowerShell
Rubeus.exe kerberoast /format:hashcat /outfile:hashes.txt
Rubeus.exe asreproast /format:hashcat /outfile:hashes.txt
Rubeus.exe s4u /user:svc_account /rc4:<Hash> /impersonateuser:Administrator /msdsspn:"CIFS/DC01" /ptt
```

### Advantages
- Comprehensive Kerberos toolkit.
- Written in C#, allowing for in-memory execution and evasion.
- Supports advanced ticket manipulation and delegation attacks.

### Limitations
- Requires execution on a Windows machine.
- C# execution can be monitored by EDR solutions.
- Complex to use for advanced attacks without deep Kerberos knowledge.

### Detection Considerations
- Monitor for Event ID 4768 (TGT requested) and 4769 (ST requested) anomalies.
- Look for the execution of Rubeus binaries or reflection-based execution.
- Monitor for unusual Kerberos ticket lifetimes or encryption types.

## Mimikatz

### Purpose
Mimikatz is a post-exploitation tool that allows users to extract plaintext passwords, hashes, PIN codes, and Kerberos tickets from memory. It can also be used to create Golden Tickets, Silver Tickets, and perform DCSync attacks.

### Installation
- Download the latest release from GitHub or compile the source code.

### Features
- LSASS memory dumping and credential extraction.
- SAM database dumping.
- NTDS.dit extraction (DCSync).
- Golden Ticket and Silver Ticket creation.
- Skeleton Key injection.

### Usage
```text
# Mimikatz
mimikatz # privilege::debug
mimikatz # sekurlsa::logonpasswords
mimikatz # lsadump::dcsync /domain:contoso.com /user:krbtgt
mimikatz # kerberos::golden /user:Administrator /domain:contoso.com /sid:<SID> /krbtgt:<Hash> /id:500 /ptt
```

### Advantages
- The gold standard for credential extraction and manipulation.
- Extremely powerful and versatile.
- Supports a wide range of offensive techniques.

### Limitations
- Highly detected by antivirus and EDR solutions.
- Requires high privileges (e.g., SeDebugPrivilege) for many features.
- Writing to disk is easily detected; in-memory execution is preferred.

### Detection Considerations
- Monitor for Event ID 10 (Sysmon) for process access to `lsass.exe`.
- Look for the execution of Mimikatz binaries or reflection-based execution.
- Monitor for Event ID 4662 (DCSync operations).

## Impacket

### Purpose
Impacket is a suite of Python classes for working with network protocols. It is widely used for SMB, NTLM, Kerberos, and LDAP attacks.

### Installation
- Install via pip: `pip3 install impacket`

### Features
- SMB client and server (smbclient.py, smbserver.py).
- NTLM relay (ntlmrelayx.py).
- Kerberos attacks (GetUserSPNs.py, GetNPUsers.py).
- DCSync (secretsdump.py).
- Remote execution (psexec.py, wmiexec.py).

### Usage
```bash
# Using Impacket
python3 GetUserSPNs.py -request -dc-ip <DC_IP> -outputfile hashes.txt contoso.com/User1:Password1
python3 secretsdump.py contoso.com/User1:Password1@<DC_IP>
python3 psexec.py contoso.com/User1:Password1@<Target_IP>
```

### Advantages
- Comprehensive suite of network protocol tools.
- Written in Python, making it cross-platform (Linux, macOS, Windows).
- Highly customizable and scriptable.

### Limitations
- Requires Python and the Impacket library to be installed.
- Network-based tools can be detected by network monitoring (IDS/IPS).
- Some features require high privileges or specific configurations.

### Detection Considerations
- Monitor for SMB traffic to unusual ports or from unusual sources.
- Look for NTLM relay attempts (Event ID 4624 with Logon Type 3).
- Monitor for LDAP queries generated by Impacket tools.

## CrackMapExec (CME) / NetExec (NXC)

### Purpose
CrackMapExec (now NetExec) is a network service exploitation tool that automates the assessment of large Active Directory environments.

### Installation
- Install via pip or package managers: `pip3 install crackmapexec` or `pip3 install netexec`

### Features
- Network enumeration (users, shares, sessions).
- Password spraying and brute force.
- Pass-the-Hash and Pass-the-Ticket.
- Remote execution (SMB, WMI, WinRM).
- Module support for various tasks.

### Usage
```bash
# Using CrackMapExec / NetExec
crackmapexec smb <Target_IP> -u User1 -p Password1
crackmapexec smb <Target_IP> -u User1 -H <NTLM_Hash>
crackmapexec smb <Target_IP> -u User1 -p Password1 -x "whoami"
```

### Advantages
- Automates enumeration and exploitation across multiple hosts.
- Supports various protocols (SMB, WinRM, SSH, LDAP, MSSQL).
- Extensible with custom modules.

### Limitations
- Network-based tool, susceptible to network monitoring.
- Can be noisy if not configured correctly (e.g., thread count).
- Requires network access to target hosts.

### Detection Considerations
- Monitor for SMB traffic from a single source to multiple destinations.
- Look for failed logon attempts (Event ID 4625) indicating password spraying.
- Monitor for remote execution events (Event ID 4697 for service installation).

## Responder

### Purpose
Responder is a tool used for LLMNR, NBT-NS, and MDNS poisoning on local networks. It captures NTLM hashes from victims who attempt to resolve non-existent hostnames.

### Installation
- Download from GitHub and install dependencies: `sudo pip3 install -r requirements.txt`

### Features
- LLMNR, NBT-NS, and MDNS poisoning.
- NTLM hash capture (via HTTP, SMB, LDAP, etc.).
- WPAD proxy attack.
- SMB relay support.

### Usage
```bash
# Using Responder
sudo responder -I eth0 -w -r -d -F
```

### Advantages
- Effective for capturing NTLM hashes in mixed environments.
- Easy to use and configure.
- Supports various protocols for hash capture.

### Limitations
- Only works on local networks (broadcast/multicast).
- Easily detected if LLMNR/NBT-NS monitoring is enabled.
- Requires the victim to attempt to resolve a non-existent hostname.

### Detection Considerations
- Monitor for DNS queries that fail, followed by LLMNR/NBT-NS broadcasts.
- Look for responses to LLMNR/NBT-NS from non-authorized IP addresses.
- Monitor for SMB traffic to unexpected destinations.

## Inveigh

### Purpose
Inveigh is a PowerShell-based tool for LLMNR, NBT-NS, and MDNS poisoning, as well as NTLM relay. It is similar to Responder but runs on Windows.

### Installation
- Download the PowerShell script or binary from GitHub.

### Features
- LLMNR, NBT-NS, and MDNS poisoning.
- NTLM hash capture.
- NTLM relay.
- HTTP and HTTPS poisoning.

### Usage
```powershell
# PowerShell
Invoke-Inveigh -ConsoleOutput Y -IP 192.168.1.100 -NBNS Y -LLMNR Y -DHCPv6 Y -Challenge 0000000000000000
```

### Advantages
- Runs natively on Windows (PowerShell or binary).
- Can be loaded into memory to avoid writing to disk.
- Supports NTLM relay on Windows.

### Limitations
- PowerShell-based, making it susceptible to AMSI and logging.
- Requires execution on a Windows machine.
- Can be noisy if not configured correctly.

### Detection Considerations
- Monitor for PowerShell script block logging (Event ID 4104).
- Look for the loading of Inveigh functions or modules.
- Monitor for LLMNR/NBT-NS responses from the attacker's IP.

## Certipy

### Purpose
Certipy is a Python tool for abusing Active Directory Certificate Services (AD CS). It is used for enumeration, exploitation, and authentication using certificates.

### Installation
- Install via pip: `pip3 install certipy-ad`

### Features
- AD CS enumeration (finding CAs and templates).
- Exploitation of ESC1, ESC2, ESC3, ESC4, ESC6, ESC8, ESC9, ESC10.
- Certificate request and authentication.
- Shadow Credentials attacks.

### Usage
```bash
# Using Certipy
python3 certipy find -username 'User1' -password 'Password1' -dc-ip <DC_IP> -target <CA_IP>
python3 certipy req -ca '<CA_Name>' -template 'VulnerableTemplate' -upn 'administrator@contoso.com'
python3 certipy auth -pfx 'administrator.pfx' -dc-ip <DC_IP>
```

### Advantages
- Comprehensive AD CS toolkit.
- Written in Python, making it cross-platform.
- Supports a wide range of AD CS attacks and Shadow Credentials.

### Limitations
- Requires Python and the Certipy library to be installed.
- Network-based tool, susceptible to network monitoring.
- Requires knowledge of AD CS concepts and templates.

### Detection Considerations
- Monitor for LDAP queries targeting AD CS attributes.
- Look for certificate requests with unusual SANs or template names.
- Monitor for authentication using certificates from unexpected sources.

### Installation
- Download the latest release from GitHub or compile the source code using Visual Studio.

### Features
- Local user and group enumeration.
- Service and scheduled task enumeration.
- Registry and file system enumeration.
- Credential and token enumeration.

### Usage

### Advantages
- Comprehensive local enumeration.

## PsExec

### Purpose
PsExec is a legitimate Sysinternals tool that allows users to execute processes on remote systems. It is frequently abused by attackers for lateral movement.

### Installation
- Download from Microsoft Sysinternals website.

### Features
- Remote process execution.
- Interactive command prompts on remote systems.
- Supports passing credentials (username/password or hash).

### Usage
```cmd
# Command Prompt
psexec.exe \\<Target_IP> -u <Domain>\<Username> -p <Password> cmd.exe
psexec.exe \\<Target_IP> -u <Domain>\<Username> -h <NTLM_Hash> cmd.exe
```

### Advantages
- Legitimate tool, making it harder to detect (living off the land).
- Easy to use for remote execution.
- Supports passing credentials or hashes.

### Limitations
- Creates a service (`PSEXESVC`) on the remote system, which can be detected.
- Requires SMB access to the remote system.
- Can be noisy if not used carefully.

### Detection Considerations
- Monitor for the creation of the `PSEXESVC` service on remote systems.
- Look for the execution of `psexec.exe`.
- Monitor for SMB traffic to remote systems.

## Evil-WinRM

### Purpose
Evil-WinRM is a tool that allows users to connect to remote Windows systems using the WinRM (Windows Remote Management) protocol. It is used for lateral movement and remote execution.

### Installation
- Install via gem: `gem install evil-winrm`

### Features
- Interactive shell on remote Windows systems.
- Supports passing credentials (username/password or hash).
- File upload and download capabilities.
- Script execution (PowerShell, batch).

### Usage
```bash
# Using Evil-WinRM
evil-winrm -i <Target_IP> -u <Username> -p <Password>
evil-winrm -i <Target_IP> -u <Username> -H <NTLM_Hash>
```

### Advantages
- Provides an interactive shell, similar to SSH.
- Easy to use for remote execution and file transfer.
- Supports passing credentials or hashes.

### Limitations
- Requires WinRM to be enabled and accessible on the remote system.
- Can be detected by network monitoring (WinRM traffic).
- Requires knowledge of WinRM configuration.

### Detection Considerations
- Monitor for WinRM traffic (ports 5985/5986) from unexpected sources.
- Look for the execution of `winrm` or `evil-winrm` commands.
- Monitor for remote execution events on the target system.

## Kerbrute

### Purpose
Kerbrute is a tool used to enumerate valid Active Directory users and perform password spraying attacks against Kerberos pre-authentication.

### Installation
- Download the latest release from GitHub.

### Features
- User enumeration (brute force usernames).
- Password spraying (brute force passwords against multiple users).
- Uses Kerberos pre-authentication (TGT requests) to avoid generating logon events.

### Usage
```bash
# Using Kerbrute
kerbrute userenum -d contoso.com --dc <DC_IP> userlist.txt
kerbrute passwordspray -d contoso.com --dc <DC_IP> userlist.txt password123
```

### Advantages
- Uses Kerberos pre-authentication, avoiding logon events (Event ID 4624/4625).
- Fast and efficient for user enumeration and password spraying.
- Easy to use and configure.

### Limitations
- Network-based tool, susceptible to network monitoring.
- Requires network access to the Domain Controller.
- Can be detected if Kerberos logging is enabled.

### Detection Considerations
- Monitor for Event ID 4768 (Kerberos TGT requested) anomalies (e.g., requests for non-existent users).
- Look for multiple TGT requests from a single source IP.
- Monitor for password spraying patterns (e.g., single password against multiple users).
